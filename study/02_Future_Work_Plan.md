# 2. 향후 작업 계획 (Roadmap)

백엔드 API(`LectureController`, `OAuth2`) 분석을 바탕으로 프론트엔드 개발을 진행할 순서입니다.

## 1단계: 라우팅 및 기본 구조 설정
가장 먼저 앱의 뼈대를 잡습니다. 어떤 URL에 어떤 페이지를 보여줄지 결정합니다.

*   **작업 내용**: `react-router-dom` 설정
*   **예상 URL 구조**:
    *   `/`: 메인 페이지 (강의 캘린더)
    *   `/login`: 로그인 페이지
    *   `/login-success`: 소셜 로그인 성공 후 토큰 처리 페이지
    *   `/lectures`: 강의 목록 페이지
    *   `/lectures/new`: 강의 등록 페이지

## 2단계: 인증(Auth) 구현 - 로그인
백엔드의 OAuth2 소셜 로그인과 연동합니다.

*   **작업 내용**:
    1.  **로그인 페이지**: "Google로 로그인" 버튼 생성 (`<a href="...">` 태그로 백엔드 엔드포인트 연결)
    2.  **로그인 성공 처리**: 백엔드에서 리다이렉트해준 URL(`login-success?token=...`)에서 **JWT 토큰**을 추출합니다.
    3.  **토큰 저장**: 추출한 토큰을 브라우저 저장소(localStorage 등)에 저장하고, 이후 API 요청마다 헤더에 자동으로 실어 보내도록 설정합니다 (Axios Interceptor).

## 3단계: 메인 기능 - 강의 캘린더
사용자가 가장 먼저 보게 될 대시보드입니다.

*   **작업 내용**:
    1.  **FullCalendar 라이브러리 설치**: 달력 UI를 쉽게 구현해주는 라이브러리입니다.
    2.  **API 연동**: `GET /api/v1/lecture/events`를 호출하여 받아온 데이터를 달력에 표시합니다.

## 4단계: 강의 관리 (CRUD)
강의를 목록으로 보고, 새로 등록하는 기능입니다.

*   **작업 내용**:
    1.  **강의 목록**: `GET /api/v1/lecture` API를 호출하여 테이블 형태로 보여줍니다.
    2.  **강의 등록 폼**: `react-hook-form`을 사용하여 입력 폼을 만들고, `POST /api/v1/lecture` API로 데이터를 전송합니다.

---

## 백엔드 개발자를 위한 팁

*   **프론트엔드는 "상태(State)" 관리가 핵심입니다.** 데이터가 변하면 화면이 자동으로 바뀝니다. (jQuery처럼 직접 DOM을 조작하지 않습니다.)
*   **비동기 처리**: API 호출은 비동기로 이루어집니다. 데이터가 오기 전(`isLoading`), 성공(`isSuccess`), 실패(`isError`) 상태에 따라 화면을 다르게 그려줘야 합니다. 이를 `TanStack Query`가 도와줍니다.
